# Contribution Guidelines

PRs are welcome. We recommend keeping each MCP "small and specialized" and ensuring it is easy to verify.

## Create a New MCP

1. Fork this repository and create a new branch.
2. Add your recommended MCP server information to the **MCP Server List** section in `README.md`.
3. Ensure detailed Description, Status, and Documentation are included.
4. Submit a Pull Request.

## Quality Bar

- Clear structure: explain "what/when/how", and include executable steps and examples.
- No sensitive information: keys, tokens, internal URLs, customer data, etc.
- Prefer reusable structured outputs (templates, checklists, fixed formats).

## PR Checklist

- MCP name and description exist.
- Includes at least 1 usage example.
- Use relative paths if referencing files within the directory.
- License is compatible with this repository's LICENSE (or explicitly includes a separate License within the MCP).
- Update the **MCP Server List** in `README.md` and `README.zh-CN.md` with your new MCP.
